Tecnologias

  • Aprendi Python há alguns anos, pois era uma linguagem que estava em tendência nos anúncios do YouTube. Eu gosto de como Python preza a simplicidade na escrita de código, mas o desempenho de seus programas sofre em contrapartida. De qualquer forma, eu tenho uma boa noção sobre essa linguagem de programação.

  • Java foi a primeira linguagem de programação que conheci. Aprendi sobre Java no meu primeiro ano no curso de Informática. Através do Java, aprendi o básico sobre lógica de programação e, eventualmente, sobre POO. Existem críticas ao Java em relação a uma falta de flexibilidade para fazer códigos de nível mais baixo, mas a linguagem ainda é útil e popular no mercado, e eu tenho um bom domínio sobre ela.

  • Apesar do meu conhecimento sobre C ainda ainda não ser tão bom, esta linguagem é considerada por muitos como uma linguagem de programação de "médio nível", pois consegue se comunicar com o desenvolvedor humano ao mesmo tempo que permite uma programação muito próxima à linguagem de máquina, assim como maior flexibilidade para utilizar os recursos do programa com melhor otimização. Apesar de ser uma linguagem complexa de se dominar, a maioria das linguagens modernas deriva de C, o que torna seu aprendizado muito útil em outras linguagens também.

  • Minha experiência com C++ também não é muito vasta, mas como aprendi C antes de C++, além de já entender sobre POO (uma novidade do C++ em relação ao C), eu consigo compreender com facilidade códigos escritos em C++. Esta, por sua vez, é uma linguagem muito adotada em desenvolvimento de sistemas que demandam otimização, como videogames, pois é mais simples de se entender que o C e ainda tem suas ferramentas que permitem otimização.

  • Kotlin é uma linguagem que eu conheci recentemente em um bootcamp e que eu particularmente achei bem interessante. É uma linguagem que deriva de Java e que permite o uso de suas bibliotecas, mas que ao mesmo tempo busca uma legibilidade na sua estrutura que permita reduzir estruturas de código em poucas linhas, o uso de funções anônimas, o uso nativo de listas e mapas, a flexibilidade em declaração de variáveis enquanto preserva a tipagem, o tratamento facilitado de valores nulos, etc. Apesar do meu domínio nessa linguagem não ser pleno, o fato de ser derivada de Java me ajuda muito a entender códigos em Kotlin.

  • Dart é a linguagem de programação mais recente no meu repertório. Como também é uma linguagem derivada de C, foi muito fácil para mim absorver as especificidades do Dart no que diz respeito à linguagem de programação. Na minha experiência inicial com Dart, eu vejo que é uma linguagem bastante semelhante ao Kotlin, apesar de não derivar de Java, pois é uma linguagem que também busca ser flexível em leitura e declaração de variáveis, ao mesmo tempo que também oferece fácil tratamento de valores nulos. Contudo, o principal diferencial do Dart certamente é o seu framework Flutter, que abordarei depois.

  • HTML é uma linguagem de marcação muito utilizada no desenvolvimento front-end. A esmagadora maioria da World Wide Web for construída utilizando HTML, que é responsável por marcar todos os elementos de uma página utilizando tags e exibí-los conforme suas configurações exigem. A versão mais recente do HTML, o HTML5, conta com tags semânticas e outras ferramentas que auxiliam a interpretação do site por ferramentas de acessibilidade. É muito importante que as páginas sejam acessíveis e bem estruturadas para garantir a retenção do público.

  • CSS é uma folha de estilo responsável por agir com o HTML e permitir a costumização dos elementos marcados nele. Com o CSS, é possível definir o estilo de uma ou múltiplas páginas e torná-las agradáveis para os usuários e responsivas, ou seja, adaptadas para diferentes aparelhos. Considerando o grande número de usuários que usam as páginas pelos celulares, é interessante pensar em responsividade para garantir que o aplicativo ou site funcione bem em telas menores.
